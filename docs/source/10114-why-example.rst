為何這個專案適合作為範例？
======================

- 示範 Swerve 全向輪的基本控制流程
- 展示多重按鍵模式與命令序列組合
- 包含視覺對齊與自動模式切換的起始架構
- 使用 WPILib 的 Command Scheduler 來管理機器人行為

學習重點
~~~~~~~~

1. **Command-Based 架構**：理解如何將機器人功能模組化為子系統和命令
2. **Swerve 驅動控制**：學習全向輪的速度控制和運動學轉換
3. **按鍵綁定**：掌握如何將操縱桿輸入映射到具體動作
4. **狀態管理**：了解如何在不同模式間切換（Coral/Algae）
5. **視覺整合**：認識 Limelight 在目標對齊中的應用

適用對象
~~~~~~~~

這個範例適合已經熟悉 Java 基礎和 WPILib 入門的學習者，希望深入了解 Command-Based 程式設計模式的隊伍。
